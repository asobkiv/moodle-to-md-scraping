# moodle-to-md-scraping

Тягне з курсу KSE Moodle усі навчальні файли (`pdf`, `docx`, `pptx`) і збирає
їх в один структурований `.md` файл на курс — щоб потім згодувати Claude як
контекст.

> Хочеш просто скачати матеріали свого курсу, без деталей реалізації? Дивись
> покрокову інструкцію в [**HOWTO.md**](HOWTO.md).

Проміжні бінарники (pdf/docx/pptx) ніколи не пишуться на диск — усе
завантажується в пам'ять і одразу конвертується. На диск лягає тільки
готовий `.md` файл у `output/`. Ніякого сервера — просто скрипт, який можна
запускати або локально, або (за замовчуванням) через GitHub Actions, щоб
креденшали не лежали на комп'ютері взагалі.

Ніщо не синкається "оптом" — потрібен явний `--course-id`. Немає його — скрипт
нічого не робить (крім `--list`).

Аутентифікація на Moodle перевикористовує підхід з
[`kse-notion-sync/moodle-to-notion`](https://github.com/asobkiv/kse-notion-sync):
спершу пробує mobile token API, інакше логіниться сесією + AJAX API.

## Запуск через GitHub Actions (креденшали лише в GitHub Secrets)

1. У репозиторії → Settings → Secrets and variables → Actions → New repository
   secret, додай:
   - `MOODLE_USERNAME`
   - `MOODLE_PASSWORD`
   - `MOODLE_BASE_URL` (опціонально, за замовчуванням `https://teaching.kse.org.ua`)

   Роби це у веб-інтерфейсі GitHub або своїм терміналом (`gh secret set
   MOODLE_PASSWORD`) — не через чат з асистентом, щоб пароль ніде не
   засвітився в історії розмови.

2. Actions → "Moodle course → Markdown" → Run workflow → введи `course_id`
   курсу (це число з URL курсу в Moodle, `?id=3261`).

3. Після завершення прогону в самому запуску буде артефакт
   `moodle-md-<course_id>` — zip з готовим `.md` файлом, завантажується
   вручну з GitHub UI. Нічого не комітиться і не лежить довше 7 днів.

## Локальний запуск (альтернатива)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# заповнити MOODLE_USERNAME / MOODLE_PASSWORD в .env

python main.py --list                # подивитись список курсів і їх id
python main.py --course-id 3261      # зібрати один курс
python main.py --course-id 3261,3508 # зібрати кілька курсів
```

Результат: `output/<Назва_курсу>.md` — по одному файлу на курс.

## Структура MD файлу

```
# Назва курсу
> метадані синхронізації, кількість файлів по типах

## Назва секції курсу

### 📄 Назва модуля (`файл.pdf`)
**Тип:** PDF · **Розмір:** ... · [Джерело в Moodle](...)

{конвертований вміст файлу, заголовки зсунуті на 3 рівні вглиб}
---
```

Ієрархія заголовків: курс = `#`, секція = `##`, файл = `###`, а весь вміст
файлу (заголовки слайдів, розділи docx, заголовки з pdf) зсувається на
`#### ` і глибше — так весь документ лишається одним чистим деревом
заголовків, зручним для читання моделлю.

## Компоненти

- `moodle_client.py` — авторизація на Moodle, виклики API, вивантаження
  списку файлів (`resource`/`folder` модулі, відфільтровані по розширенню),
  завантаження байтів файлу.
- `converters.py` — конвертація байтів у Markdown:
  - `pdf` → [`pymupdf4llm`](https://pypi.org/project/pymupdf4llm/) (визначає
    заголовки за розміром шрифту, зберігає таблиці)
  - `docx` → `python-docx`, обхід параграфів/таблиць у порядку документа
  - `pptx` → `python-pptx`, слайд = заголовок, буліти з рівнями відступів,
    таблиці, нотатки доповідача
- `main.py` — оркестрація: логін → курси → файли → завантаження+конвертація
  → збірка одного MD файлу на курс.

## Обмеження

- Береться вміст лише з модулів `resource` та `folder` (як і в оригінальному
  `moodle-to-notion` скрипті) — не з `assign`/`page`/тощо.
- Розширення обмежені `pdf`, `docx`, `pptx` — інші типи файлів пропускаються.
- Якщо конвертація конкретного файлу впала (пошкоджений/захищений файл),
  у підсумковому MD замість вмісту буде позначка про помилку — синхронізація
  курсу продовжується.
