# Як скачати матеріали курсу з Moodle у Markdown-файл

Інструкція для будь-кого зі своїм акаунтом на KSE Moodle. Технічних знань не
потрібно — лише GitHub-акаунт (безкоштовний) і 10 хвилин.

Кожен користувач працює **зі своєю власною копією** репозиторію і своїми
власними Moodle-креденшалами — вони нікому іншому не видні, навіть власнику
оригінального репозиторію.

---

## Крок 0. Що знадобиться

- Акаунт на [github.com](https://github.com) (якщо нема — зареєструватись,
  безкоштовно)
- Логін і пароль від твого KSE Moodle (той самий, яким заходиш на
  `teaching.kse.org.ua`)

---

## Крок 1. Скопіювати репозиторій собі ("форкнути")

1. Відкрий https://github.com/asobkiv/moodle-to-md-scraping
2. Вгорі праворуч натисни кнопку **Fork**
3. На наступній сторінці нічого не міняй, натисни **Create fork**

Тепер у тебе є власна копія за адресою
`https://github.com/ТВІЙ_НІК/moodle-to-md-scraping`.

---

## Крок 2. Додати свої Moodle-креденшали як "секрети"

Це приватне сховище паролів усередині GitHub — його ніхто, включно з
власником оригінального репозиторію, не бачить і не може прочитати.

1. У своєму форку відкрий вкладку **Settings** (вгорі)
2. Зліва в меню: **Secrets and variables** → **Actions**
3. Натисни зелену кнопку **New repository secret**
4. Заповни:
   - **Name:** `MOODLE_USERNAME`
   - **Secret:** твій логін/email для Moodle
   - Натисни **Add secret**
5. Повтори ще раз для пароля:
   - **Name:** `MOODLE_PASSWORD`
   - **Secret:** твій пароль від Moodle
   - Натисни **Add secret**

(Секрет `MOODLE_BASE_URL` не потрібен, якщо навчаєшся на
`teaching.kse.org.ua` — це значення за замовчуванням.)

---

## Крок 3. Увімкнути автозапуски (Actions) у своєму форку

GitHub вимикає їх за замовчуванням у форкнутих репозиторіях.

1. Відкрий вкладку **Actions**
2. З'явиться жовте попередження "Workflows aren't being run on this forked
   repository"
3. Натисни кнопку **I understand my workflows, go ahead and enable them**

---

## Крок 4. Знайти ID потрібного курсу в Moodle

1. Залогінься на https://teaching.kse.org.ua
2. Відкрий сторінку потрібного курсу
3. Подивись на адресний рядок браузера — там буде щось на кшталт:
   `.../course/view.php?id=3949`
4. Число після `id=` — це і є `course_id` (у прикладі — `3949`)

---

## Крок 5. Запустити вивантаження

1. У своєму форку на GitHub відкрий вкладку **Actions**
2. Зліва в списку workflow-ів обери **Moodle course → Markdown**
3. Праворуч з'явиться кнопка **Run workflow** — натисни її
4. У полі **course_id** введи число з Кроку 4
5. Натисни зелену кнопку **Run workflow**

---

## Крок 6. Дочекатись завершення

1. Онови сторінку через ~10 секунд — з'явиться новий рядок запуску
2. Жовте коло 🟡 = ще виконується, зазвичай 1–3 хвилини залежно від
   кількості й розміру файлів у курсі
3. Зелена галочка ✅ = готово

---

## Крок 7. Забрати готовий файл — і де він опиниться

1. Клацни на завершений (✅) запуск у списку
2. Прогорни сторінку запуску вниз до розділу **Artifacts**
3. Натисни на файл з назвою на кшталт `moodle-md-3949` — почнеться
   завантаження **zip-архіву**
4. Архів прилетить у стандартну папку завантажень браузера — зазвичай:
   - **Downloads** (macOS/Windows) або **Завантаження**
5. Розпакуй zip (подвійний клік по архіву). Всередині — один `.md` файл з
   назвою курсу, напр. `MGMT301_Operations_Management_BE27.md`

Це і є фінальний файл — його можна відкрити будь-яким текстовим редактором
або одразу віддати Claude як контекст для роботи з матеріалами курсу.

---

## Якщо щось не спрацювало

| Симптом | Причина / рішення |
|---|---|
| Немає кнопки **Run workflow** | Actions ще не увімкнені — повернись до Кроку 3 |
| Запуск завершився ❌ (червоний) з "Could not fetch course" | Неправильний `course_id`, або ти не записаний на цей курс у Moodle |
| Запуск завершився ❌ з помилкою логіну | Перевір `MOODLE_USERNAME`/`MOODLE_PASSWORD` в Secrets — можливо, одруківка чи застарілий пароль |
| Немає розділу **Artifacts** | Або запуск ще не завершився, або впав з помилкою до вивантаження файлів |
| У курсі 0 файлів у результаті | У курсі просто немає файлів `pdf`/`docx`/`pptx` в модулях "Файл"/"Папка" (інші типи матеріалів — сторінки, посилання, завдання — свідомо пропускаються) |
